# 用户手册

## 1. 产品用途

Moto 用于记录摩托车的日常使用信息，重点覆盖：

- 车辆档案
- 加油记录
- 保养记录
- 首页费用与油耗概览
- 账户密码管理

适合个人车主或少量账号共同使用。

## 2. 首次使用

### 2.1 打开系统

浏览器访问部署地址，例如：

- 本地开发：`http://127.0.0.1:18080`
- Docker 映射：按实际映射端口访问

### 2.2 注册或登录

登录页支持：

- 注册新账号
- 使用已有账号登录

登录成功后会自动进入首页，并把登录态保存在当前浏览器。

## 3. 页面说明

### 3.1 仪表盘

仪表盘是默认进入页面，主要展示默认车辆的关键状态。

可看到的内容包括：

- 本月总支出
- 总支出
- 最新油耗 / 平均油耗
- 平均油价
- 最近一次加油
- 最近一次保养
- 下次保养提醒
- 油耗趋势

说明：

- 默认车辆由你在“车辆”页手动切换
- 如果没有设置默认车辆，系统会优先使用第一台车辆

### 3.2 加油

用于维护加油记录。

支持操作：

- 查看记录列表
- 按车辆过滤
- 新增记录
- 编辑记录
- 删除记录

字段说明：

- 日期：加油日期
- 车辆：本次加油属于哪台车
- 里程：加油时里程
- 升数：本次加油量
- 单价：每升单价
- 总价：本次总费用
- 是否满油：区分满油/未满油
- 备注：补充说明

建议：

- 里程尽量按真实顺序录入
- 如果希望油耗计算更可信，尽量保持记录完整

### 3.3 保养

用于维护保养和维修记录。

支持操作：

- 查看记录列表
- 按车辆过滤
- 新增记录
- 编辑记录
- 删除记录

字段说明：

- 日期：保养日期
- 车辆：本次保养属于哪台车
- 保养项目：本次做了什么
- 费用：本次保养支出
- 本次里程：保养时里程
- 下次保养里程：下次计划处理的里程点
- 备注：补充说明

项目录入方式：

- 可以直接手写
- 也可以点击系统预置常用项，自动追加到标题中

首页提醒规则：

- 系统读取默认车辆最近一条保养记录
- 如果填写了“下次保养里程”，首页会根据当前里程给出提示

### 3.4 车辆

用于管理车辆档案。

支持操作：

- 新增车辆
- 编辑车辆
- 删除车辆
- 设置默认车辆

字段说明：

- 名称：车辆显示名，建议必填
- 品牌：例如豪爵、雅马哈
- 型号：例如 UHR150
- 排量：例如 150cc
- 购入日期：购车日期
- 初始里程：车辆初始或当前基准里程
- 备注：补充说明

默认车辆作用：

- 仪表盘统计围绕默认车辆
- 最近记录与保养提醒围绕默认车辆

删除限制：

- 如果该车已有加油或保养记录，系统不允许删除

### 3.5 我的

用于查看账户概况和做安全操作。

包含内容：

- 用户名
- 车辆数量
- 默认车辆
- 加油记录数
- 保养记录数

可执行操作：

- 修改自己的密码
- 管理员重置其他用户密码
- 退出登录

## 4. 常见业务说明

### 4.1 默认车辆

默认车辆是首页统计和提醒的基准。

切换方式：

1. 进入“车辆”
2. 找到目标车辆
3. 点击“设为默认”
4. 确认切换

### 4.2 油耗理解

当前系统会根据加油记录计算油耗相关数据，前提是：

- 记录中里程递增
- 数据尽量连续

如果只有一条记录，或相邻记录里程异常，油耗值可能为 0 或没有参考意义。

### 4.3 保养提醒理解

提醒基于两部分：

- 当前车辆里程
- 最近一次保养记录中的“下次保养里程”

因此如果你希望首页准确提醒，需要在保养记录里填好“下次保养里程”。

## 5. 推荐使用流程

建议按这个顺序使用：

1. 先新增车辆
2. 设置默认车辆
3. 录入历史加油记录
4. 录入历史保养记录
5. 后续持续更新加油和保养
6. 定期在首页查看费用、油耗和保养提醒

## 6. 常见问题

### 6.1 为什么首页没有统计？

常见原因：

- 没有车辆
- 没有设置默认车辆
- 默认车辆还没有记录

### 6.2 为什么不能删除车辆？

因为该车辆已经有关联记录：

- 加油记录
- 保养记录

需要先清理关联记录，才能删除车辆。

### 6.3 为什么首页没有保养提醒？

常见原因：

- 默认车辆没有保养记录
- 最近一条保养记录没有填写下次保养里程

### 6.4 管理员重置密码后，新密码是什么？

当前固定为：

- `moto123`

## 7. 注意事项

- 默认车辆设置保存在当前浏览器，本机清缓存后可能丢失
- 当前项目部分中文文案仍有乱码，属于已知问题
- 如果使用 Docker 部署，务必把数据库文件做外部挂载
